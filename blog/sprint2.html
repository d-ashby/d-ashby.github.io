What are the differences between static, relative, absolute, and fixed positioning?
20 April 2022

position:static; Websites by default have elements position themselves statically, this means that they are laid out how they are laid out in the html document. The order of the elements that are added to the HTML file is the order that they are displayed via the browser (chrome, firefox, safari).


position:relative; or Relative positioning is moving an element in relation to where it supposed to be statically. For example if an element is specified to have
position:relative; top:150px; left:50px;
The box specified moves 150 pixels from the top, and 50 pixels from the left, in other words its moved down to the right or south-east.

Position:absolute; or absolute positioning, changes an elements position based on the container I'ts in. I'ts the same in the sense that you move an element using top, left, right, bottom - however it moves the element not based on where it was supposed to be (relative positioning) but move based on the container(in this instance its the webpage acting as the container). This absolute positioning removes the element from the flow of static elements therefore if you have following elements they will then 'move up the list' and take the position of the element that you have moved using absolute positioning. See below how the blue violet moves up to the coral boxes original static position.

position:fixed; or fixed positioning, causes the element to stay in a position relative to window of the browser (viewport). For example 'position:fixed; top:0;' would be useful to always have a banner position itself at the top of the page. despite scrolling this fixed position never changes because it is fixed to the window not the webpage.
